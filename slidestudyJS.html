Module 02 — Cú Pháp Cơ Bản
Biến: var · let · const
Cách JavaScript lưu trữ và quản lý dữ liệu — hiểu rõ sự khác biệt là bước đầu quan trọng nhất.
variables.jsJS 
// ✅ const — không thể gán lại (dùng mặc định)
const appName = "StayZ";
const maxRooms = 100;
// appName = "other"; ← ❌ TypeError!

// ✅ let — có thể thay đổi giá trị
let currentPage = 1;
currentPage = 2; // ← ✅ OK

// ⚠️  var — tránh dùng (vấn đề về scope)
var oldStyle = "avoid this";

// Khai báo nhiều biến cùng lúc
const { name, price, rating } = hotel;
let [first, ...rest] = rooms;
⚠️ var hoisting: var được "kéo lên đầu" scope, có thể dùng trước khi khai báo — gây bug khó tìm. Hãy dùng let và const thay thế.
So sánh var · let · const
Từ khóa Gán lại Block Scope Hoisting
var ✅ ❌ (function) ⚠️ undefined
let ✅ ✅ ❌ TDZ error
const ❌ ✅ ❌ TDZ error
Quy tắc ngón tay cái: Luôn dùng const. Chỉ chuyển sang let khi bạn biết rõ giá trị cần thay đổi. Không dùng var.
📌 TDZ — Temporal Dead Zone
console.log(x); // ReferenceError
let x = 5;
let/const tồn tại trong TDZ trước dòng khai báo — truy cập sẽ throw lỗi ngay lập tức, tốt hơn var.
📏
Quy tắc 95% trường hợp: dùng const. Cần thay đổi → let. Không bao giờ dùng var trong code mới.
Module 02 — Cú Pháp Cơ Bản
Kiểu Dữ Liệu & Type Coercion
JS có 8 kiểu dữ liệu — và hành vi "ép kiểu tự động" là nguồn gốc của nhiều bug bí ẩn.
types.jsJS 
// Primitive types
const name  = "Huy";          // string
const age   = 22;             // number
const score = 9.5;            // number (float)
const isAdm = true;          // boolean
const phone = null;          // null (có chủ ý)
let   draft;                  // undefined (chưa gán)

// Reference types
const user    = { name, age }; // object
const rooms   = [101, 102];   // array
const greet   = () => "hi";  // function

// Kiểm tra kiểu
typeof name    // "string"
typeof rooms   // "object" ⚠️ (not "array"!)
Array.isArray(rooms) // true ✅
⚠️ Type Coercion — Bẫy Phổ Biến
coercion.jsJS 
// == (loose equality) — TỰ ÉP KIỂU
0  ==  ""      // true  😱
null == undefined // true  😱
"5" ==  5      // true  😱

// === (strict equality) — KHÔNG ÉP KIỂU ✅
"5" === 5      // false ✅
null === undefined // false ✅

// Cộng string + number
"10" + 5    // "105" 😱 (concatenation!)
10   - "5"  // 5   ✅ (subtraction forced)

// Chuyển kiểu đúng cách
Number("42")   // 42
String(42)    // "42"
Boolean(0)    // false
Quy tắc: Luôn dùng === thay vì ==. Tránh phụ thuộc vào type coercion ngầm định.
⚠️
Dùng === (strict) thay  ==  (loose). Khi cộng số từ input HTML, luôn Number(value) trước — input trả về string!
Module 03 — Functions
Functions: 3 Cách Viết
Function là khối xây dựng cơ bản của mọi chương trình JS. Hiểu sự khác biệt giúp viết code sạch hơn.
functions.jsJS 
// 1️⃣ Function Declaration — có hoisting
function greet(name) {
  return `Xin chào, ${name}!`;
}

// 2️⃣ Function Expression — không có hoisting
const add = function(a, b) {
  return a + b;
};

// 3️⃣ Arrow Function — cú pháp ngắn gọn ✅
const multiply = (a, b) => a * b;

const square = n => n * n; // 1 tham số → bỏ ()

const getUser = id => ({    // return object
  id,
  url: `/users/${id}`
});

// Default parameters
const createRoom = (num, type = "standard") => ({
  num, type
});
So Sánh — Khi Nào Dùng Gì?
Loại Hoisting this Dùng cho
Declaration ✅ Có Dynamic Hàm chính, utilities
Expression ❌ Dynamic Callback, module
Arrow ❌ Lexical ✅ Callback, array methods
Arrow function và this: Arrow không tạo this riêng — kế thừa từ scope bên ngoài. Cực kỳ hữu ích khi dùng trong event listener hoặc callback.
Kết quả
greet("Huy") → "Xin chào, Huy!"
multiply(3, 4) → 12
square(5) → 25
getUser(1) → { id: 1, url: "/users/1" }
✨
Trong code hiện đại: dùng arrow function cho callbacks và array methods. Dùng function declaration cho hàm chính của module.
Module 03 — Functions
Scope & Closure
Scope quyết định biến nào "nhìn thấy" được ở đâu. Closure là superpower của JavaScript.
scope.jsJS 
// Global scope — dùng được mọi nơi
const API_URL = "https://api.stayz.vn";

function fetchRooms() {
  // Function scope
  const endpoint = "/rooms";

  if (true) {
    // Block scope (let/const)
    let temp = "chỉ trong if này";
    console.log(API_URL);  // ✅ global ok
    console.log(endpoint);  // ✅ outer ok
  }
  // console.log(temp); ← ❌ ReferenceError
}
closure.jsJS 
// Closure — hàm nhớ scope nơi nó được tạo
function makeCounter() {
  let count = 0; // biến private!

  return {
    inc: () => ++count,
    dec: () => --count,
    get: () => count
  };
}

const counter = makeCounter();
counter.inc(); // 1
counter.inc(); // 2
counter.get(); // 2 ← count không bị reset!
Scope Chain — Visual
🌐 Global Scope
API_URL, fetchRooms
📦 Function Scope (fetchRooms)
endpoint
🔒 Block Scope (if)
temp
Thấy được: temp, endpoint, API_URL
Closure trong thực tế: Event handlers, module pattern, React hooks — tất cả đều dựa trên closure. Đây là concept quan trọng nhất của JS.
Closure giúp tạo private state
count chỉ truy cập được qua inc/dec/get
Không ai có thể counter.count = 999 từ ngoài
🔒
Closure cho phép hàm "nhớ" biến từ scope nơi nó được tạo — ngay cả sau khi hàm cha đã chạy xong. Dùng để tạo private state và factory functions.
Module 04 — Control Flow
if / else · switch · Ternary
Kiểm soát luồng thực thi — biết khi nào dùng cái gì giúp code ngắn gọn và dễ đọc hơn.
conditions.jsJS 
// if / else if / else
function getRoomType(price) {
  if (price >= 500) return "Suite";
  else if (price >= 200) return "Deluxe";
  else if (price >= 100) return "Standard";
  else return "Budget";
}

// switch — khi so sánh 1 giá trị nhiều case
switch (status) {
  case "pending":  label = "⏳ Chờ"; break;
  case "confirmed":"✅ Xác nhận"; break;
  case "cancelled":"❌ Huỷ";     break;
  default: label = "❓ Không rõ";
}

// Ternary — rút gọn if/else 1 dòng
const label = isLoggedIn
  ? "Đăng xuất"
  : "Đăng nhập";

// Nullish coalescing ?? — thay thế || an toàn hơn
const name = user.name ?? "Khách";

// Optional chaining ?. — tránh TypeError
const city = user?.address?.city ?? "N/A";
Logical Operators nâng cao
logical.jsJS 
// Short-circuit evaluation
const user = null;

// || → trả về giá trị TRUTHY đầu tiên
const a = user || "Khách";  // "Khách"
// ⚠️ 0, "", false cũng bị coi là falsy!

// ?? → chỉ fallback khi null / undefined
const count = 0 ?? "N/A";  // 0 ✅
const count2 = 0 || "N/A";  // "N/A" 😱

// &&  → render điều kiện (React pattern)
isAdmin && showAdminPanel();

// Falsy values trong JS
// false, 0, "", null, undefined, NaN
?? vs ||: Dùng ?? khi giá trị 0 hoặc "" có ý nghĩa thực sự (ví dụ: số lượng, giá tiền). Dùng || khi cần fallback cho mọi falsy value.
?. Optional chaining: user?.profile?.avatar không throw lỗi nếu user hoặc profile là null/undefined — trả về undefined thay vì crash.
🧠
ES2020 thêm ?? (nullish coalescing) và ?. (optional chaining) — hai operator này giúp code an toàn hơn rất nhiều. Học thuộc ngay.
Module 04 — Control Flow
Vòng Lặp: for · while · for...of
Biết chọn đúng loại vòng lặp giúp code ngắn hơn, nhanh hơn và dễ đọc hơn.
loops.jsJS 
const rooms = ["101", "102", "103"];

// for — khi cần index hoặc đếm ngược
for (let i = 0; i < rooms.length; i++) {
  console.log(`Room ${rooms[i]}`);
}

// for...of — duyệt array sạch nhất ✅
for (const room of rooms) {
  console.log(room);
}

// for...in — duyệt keys của object
const user = { name: "Huy", age: 22 };
for (const key in user) {
  console.log(`${key}: ${user[key]}`);
}

// while — khi điều kiện dừng chưa biết trước
let attempts = 0;
while (attempts < 3) {
  tryLogin(); attempts++;
}

// break / continue
for (const room of rooms) {
  if (room === "102") continue; // bỏ qua
  if (room === "103") break;    // dừng
}
Khi Nào Dùng Gì?
Loại Dùng khi
for (i) Cần index, đếm ngược, bước nhảy ≠ 1
for...of Duyệt array, string, Set, Map
for...in Duyệt keys của plain object
while Lặp đến khi điều kiện đổi (pagination...)
forEach Side effects (log, update DOM)
⚠️ forEach không thể break/return sớm. Nếu cần dừng giữa chừng, dùng for...of hoặc for thay thế.
Performance: Với mảng lớn, for truyền thống nhanh nhất. for...of sạch và đủ nhanh cho hầu hết trường hợp thực tế.
🔄
Trong thực tế 80% dùng for...of hoặc array methods (map/filter/reduce). Dùng for(i) khi thực sự cần index.
Module 05 — Arrays
Array Methods: Biến Đổi Dữ Liệu
map · filter · reduce — ba method quan trọng nhất trong JavaScript hiện đại. Thành thạo ba cái này là đủ cho 90% công việc.
array-methods.jsJS 
const products = [
  { name: "iPhone",  price: 25000000, inStock: true },
  { name: "Samsung", price: 18000000, inStock: false},
  { name: "Pixel",   price: 22000000, inStock: true },
];

// MAP → transform mỗi phần tử, trả về array mới
const names = products.map(p => p.name);
// ["iPhone", "Samsung", "Pixel"]

const withVAT = products.map(p => ({
  ...p,
  priceWithVAT: p.price * 1.1
}));

// FILTER → lọc, trả về array con
const available = products.filter(p => p.inStock);
// [iPhone, Pixel]

const under20M = products.filter(p => p.price < 20000000);

// REDUCE → gộp thành 1 giá trị
const total = products.reduce(
  (acc, p) => acc + p.price, 0
);
// 65000000

// CHAIN — kết hợp
const totalAvail = products
  .filter(p => p.inStock)
  .reduce((acc, p) => acc + p.price, 0);
Thêm Methods Quan Trọng
more-methods.jsJS 
const arr = [1, 2, 3, 4, 5];

// find → phần tử đầu tiên thỏa điều kiện
arr.find(x => x > 3)      // 4

// findIndex → index của phần tử
arr.findIndex(x => x > 3)  // 3

// some → có ít nhất 1 phần tử thỏa?
arr.some(x => x > 4)       // true

// every → tất cả đều thỏa?
arr.every(x => x > 0)      // true

// includes → có tồn tại phần tử này?
arr.includes(3)           // true

// flat → làm phẳng mảng lồng nhau
[1, [2, 3], [4]].flat()  // [1,2,3,4]

// sort — LƯU Ý: mutates original!
[...arr].sort((a, b) => a - b)  // copy trước!
map, filter, reduce không thay đổi array gốc — luôn trả về array mới. Đây gọi là immutability, là pattern quan trọng trong React và code hiện đại.
🔗
Chain methods để xử lý dữ liệu rõ ràng: .filter(...).map(...).reduce(...) — đọc như câu lệnh tiếng Anh, dễ debug và test hơn vòng lặp lồng nhau.
Module 05 — Objects & ES6+
Objects, Destructuring & Spread
Destructuring và Spread là hai cú pháp ES6 xuất hiện nhiều nhất trong code React và Node.js thực tế.
destructuring.jsJS 
const hotel = {
  id: 1, name: "Caravelle", stars: 5,
  address: { city: "HCMC", district: "Q1" }
};

// Object destructuring
const { name, stars } = hotel;
console.log(name);   // "Caravelle"

// Rename khi destructure
const { name: hotelName, stars: rating } = hotel;

// Nested destructuring
const { address: { city } } = hotel;

// Default value
const { pool = false } = hotel; // false

// Array destructuring
const [first, second, ...rest] = [1, 2, 3, 4];
// first=1, second=2, rest=[3,4]

// Swap variables
let a = 1, b = 2;
[a, b] = [b, a]; // a=2, b=1

// Function params destructuring
const displayHotel = ({ name, stars, address: { city } }) =>
  `${name} (${stars}★) - ${city}`;
spread-rest.jsJS 
// Spread — mở rộng
const base = { a: 1, b: 2 };
const ext  = { ...base, c: 3 }; // {a,b,c}

// Update object (immutable pattern)
const updated = { ...hotel, stars: 4 };
// hotel vẫn giữ nguyên, updated có stars=4

// Clone array
const copy = [...original];

// Merge arrays
const all = [...arr1, ...arr2, 999];

// Rest params — gom phần còn lại
const sum = (...nums) =>
  nums.reduce((a, b) => a + b, 0);

sum(1, 2, 3, 4); // 10

// Object shorthand (ES6)
const name = "Huy";
const age  = 22;
const user = { name, age }; // {name:"Huy",age:22}
Spread để copy, không phải reference: {...obj} tạo shallow copy. Thay đổi obj.nested vẫn ảnh hưởng copy. Dùng structuredClone(obj) cho deep copy.
✨
Destructuring + Spread là cú pháp xuất hiện nhiều nhất trong React props, Redux reducers và API responses. Thuộc lòng pattern này.

dom-create.jsJS 
// Tạo và thêm phần tử
function createProductCard(product) {
  const card = document.createElement('div');
  card.className = 'product-card';
  card.innerHTML = `
    <img src="${product.image}" alt="${product.name}">
    <h3>${product.name}</h3>
    <p class="price">${product.price}đ</p>
    <button data-id="${product.id}">Thêm giỏ</button>
  `;
  return card;
}

// ✅ Render list hiệu quả (1 lần DOM update)
function renderProducts(list) {
  const container = document.querySelector('#products');
  container.innerHTML = list
    .map(p => `<div class="card">
      <h3>${p.name}</h3><p>${p.price}đ</p>
    </div>`)
    .join("");
}
innerHTML vs textContent: Không bao giờ inject user input qua innerHTML — dễ bị XSS. Dùng textContent cho plain text, innerHTML chỉ cho HTML template của bạn.
⚡
Render list: build HTML string bằng .map().join("") rồi gán vào innerHTML một lần — nhanh hơn nhiều so với append từng phần tử.
Module 06 — DOM
Events & Event Delegation
Event handling là trái tim của mọi web app tương tác. Event delegation là pattern không thể thiếu.
events.jsJS 
// addEventListener — cách duy nhất nên dùng
const btn = document.querySelector('#addToCart');

btn.addEventListener('click', function(event) {
  event.preventDefault(); // ngăn hành vi mặc định
  event.stopPropagation(); // ngăn bubble lên cha
  console.log("Clicked!", event.target);
});

// Các events phổ biến
el.addEventListener('click', fn);      // chuột
el.addEventListener('input', fn);      // gõ phím
el.addEventListener('change', fn);     // select/checkbox
el.addEventListener('submit', fn);     // form submit
el.addEventListener('keydown', fn);    // keyboard
el.addEventListener('mouseenter', fn); // hover in
el.addEventListener('scroll', fn);     // scroll

// Xoá event listener (cần giữ reference hàm)
const handler = () => console.log('hi');
btn.addEventListener('click', handler);
btn.removeEventListener('click', handler);
event-delegation.jsJS 
// ❌ Sai: gắn listener vào TỪNG phần tử
// (tốn memory, không hoạt động với phần tử mới)
document.querySelectorAll('.del-btn')
  .forEach(btn => btn.addEventListener('click', del));

// ✅ Đúng: Event Delegation
// Gắn 1 listener vào phần tử cha
const list = document.querySelector('#product-list');

list.addEventListener('click', (e) => {
  // Kiểm tra phần tử nào được click
  if (e.target.matches('.del-btn')) {
    const id = e.target.dataset.id;
    deleteProduct(id);
  }
  if (e.target.matches('.add-cart')) {
    const id = e.target.dataset.id;
    addToCart(id);
  }
});
Event Delegation hoạt động nhờ Event Bubbling: Click trên phần tử con sẽ "nổi bọt" lên cha, ông... Listener ở cha bắt được tất cả. Pattern này quan trọng khi render danh sách động.

validate.jsJS 
function validate(data) {
  const errors = [];
  const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;

  if (!data.name)
    errors.push("Họ tên không được để trống");

  if (!emailRegex.test(data.email))
    errors.push("Email không hợp lệ");

  if (!data.checkin)
    errors.push("Vui lòng chọn ngày");

  if (data.rooms < 1 || data.rooms > 10)
    errors.push("Số phòng từ 1–10");

  return errors;
}

function showErrors(errors) {
  const el = document.querySelector('#errors');
  el.innerHTML = errors
    .map(e => `<p class="error">❌ ${e}</p>`)
    .join("");
}
Không bao giờ chỉ validate ở client. Server phải validate lại — người dùng có thể bypass JavaScript hoàn toàn bằng cách gọi API trực tiếp.
📋
Pattern chuẩn: preventDefault → collect data → validate → show errors hoặc submit. Tách hàm validate riêng để dễ test và tái sử dụng.

api-error-handling.jsJS 
// Error handling chuẩn cho API calls
async function fetchData(url) {
  try {
    showLoading(true);
    const res = await axios.get(url);
    return res.data;

  } catch (err) {
    // Axios error có response từ server
    if (err.response) {
      const { status, data } = err.response;
      if (status === 401) redirectToLogin();
      if (status === 404) showNotFound();
      if (status === 500) showServerError();
    } else if (err.request) {
      // Request đã gửi nhưng không có response
      showNetworkError("Không có mạng");
    } else {
      console.error("Lỗi:", err.message);
    }
    throw err; // re-throw để caller biết

  } finally {
    showLoading(false);
  }
}
finally là nơi dọn dẹp: tắt loading spinner, đóng connection. Chạy dù thành công hay lỗi — không cần lặp lại trong cả try lẫn catch.
🛡️
Mỗi async call nên có try/catch. Dùng finally để dọn dẹp. Phân biệt loại lỗi (network, 4xx, 5xx) để hiển thị thông báo phù hợp cho người dùng.
Module 07 — Async
Promises: Bất Đồng Bộ Có Kiểm Soát
Promise là nền tảng của async JavaScript. Dù async/await phổ biến hơn, hiểu Promise giúp debug và dùng các API cũ tốt hơn.
promises.jsJS 
// Promise có 3 trạng thái: pending → fulfilled | rejected
const p = new Promise((resolve, reject) => {
  setTimeout(() => {
    if (Math.random() > 0.5) {
      resolve("Thành công!");
    } else {
      reject(new Error("Thất bại!"));
    }
  }, 1000);
});

// .then .catch .finally chain
p
  .then(result => console.log(result))
  .catch(err => console.error(err))
  .finally(() => console.log("Done"));

// Promise.all — chạy song song, chờ tất cả
const [hotels, rooms, reviews] = await Promise.all([
  axios.get('/hotels'),
  axios.get('/rooms'),
  axios.get('/reviews'),
]);

// Promise.allSettled — tất cả, kể cả lỗi
const results = await Promise.allSettled([p1, p2, p3]);
results.forEach(r => {
  if (r.status === 'fulfilled') use(r.value);
  else log(r.reason);
});
Promise States — Visual
pending
Đang xử lý, chưa có kết quả
│
├─── fulfilled ──→
.then(value => ...)
└─── rejected ───→
.catch(err => ...)
Promise.all vs allSettled
Method Hành vi khi 1 reject Dùng khi
all Reject ngay (fail-fast) Tất cả phải thành công
allSettled Vẫn chờ tất cả Xử lý từng kết quả riêng
race Lấy kết quả đầu tiên Timeout pattern
any Lấy fulfilled đầu tiên Fallback server
⚡
Promise.all là cách chạy nhiều API call song song — nhanh hơn nhiều so với await từng cái một. Thời gian = max(t1, t2, t3) thay vì t1 + t2 + t3.

// Render flow hoàn chỉnh
async function loadProducts() {
  const container = document.querySelector('#app');

  // 1. Show loading state
  container.innerHTML = `
    <div class="skeleton">Đang tải...</div>
  `;

  try {
    // 2. Fetch data
    const products = await getProducts();

    // 3. Render
    container.innerHTML = products
      .map(p => `
        <div class="card">
          <h3>${p.name}</h3>
          <p>${p.price.toLocaleString('vi-VN')}đ</p>
          <button data-id="${p.id}" class="buy-btn">
            Mua ngay
          </button>
        </div>
      `)
      .join("");

  } catch (err) {
    // 4. Error state
    container.innerHTML = `
      <p class="error">❌ ${err.message}</p>
      <button onclick="loadProducts()">Thử lại</button>
    `;
  }
}

  renderCart(state.cart);

filter-flow.jsJS 
import { state } from './core.js';

// Debounce — delay function call
function debounce(fn, delay = 300) {
  let timer;
  return (...args) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}

// Search với debounce
const handleSearch = debounce((query) => {
  state.searchQuery = query.toLowerCase();
  state.filteredProducts = state.products.filter(p =>
    p.name.toLowerCase().includes(state.searchQuery)
  );
  renderProductList(state.filteredProducts);
});

// Cart logic
export function addToCart(productId) {
  const existing = state.cart.find(i => i.id === productId);

  if (existing) {
    existing.qty++;
  } else {
    const product = state.products.find(p => p.id === productId);
    state.cart.push({ ...product, qty: 1 });
  }
  renderCart(state.cart);
}
Debounce quan trọng: Người dùng gõ 10 ký tự → 10 search call. Với debounce 300ms, chỉ gọi sau khi ngừng gõ 300ms → 1 call. Tiết kiệm request và CPU.
🗃️
State = nguồn sự thật duy nhất. Không update DOM trực tiếp — update state rồi gọi render(). Đây chính xác là cách React hoạt động bên trong.

10 Lỗi Junior Hay Mắc
Những lỗi này xuất hiện trong mọi dự án thực tế. Học từ sai lầm của người khác trước khi tự mắc.
❌ 1. Không await async function
const data = getUser(); // Promise, không phải data!
const data = await getUser(); // ✅
❌ 2. Mutate state trực tiếp
state.products.push(item); // ❌
state.products = [...state.products, item]; // ✅
❌ 3. Dùng == thay vì ===
if (count == "0") // "0" == 0 là true 😱
if (count === 0) // ✅
❌ 4. Không xử lý async error
await fetchData(); // nếu lỗi → crash
try { await fetchData(); } catch(e) {...} // ✅
❌ 5. Quên Number() khi lấy từ input
"5" + 3 === "53" // string!
Number(input.value) + 3 === 8 // ✅
❌ 6. var thay vì let/const
for (var i = 0; i < 3; i++) setTimeout(()=>log(i))
// In ra 3, 3, 3 — dùng let để fix!
❌ 7. Gán lại const object sai
const arr = []; arr = [1,2]; // TypeError!
const arr = []; arr.push(1); // ✅ push được
❌ 8. forEach khi cần return
arr.forEach(x => { return x*2; }); // return bị bỏ qua
arr.map(x => x*2); // ✅
❌ 9. Không kiểm tra null/undefined
user.profile.name // TypeError nếu profile null
user?.profile?.name // ✅
❌ 10. Mutate array khi sort
data.sort((a,b) => a-b); // thay đổi data gốc!
[...data].sort((a,b) => a-b); // ✅ copy trước

Minimax & Alpha-Beta Pruning
Thuật toán AI cho game cờ caro — ứng dụng thực tế của recursion, tree traversal và optimization.
caro.jsJS 
function minimax(board, depth, isMaximizing, alpha, beta) {
  // Base case: kiểm tra kết thúc game
  const score = evaluate(board);
  if (score !== null) return score;
  if (depth === 0) return heuristic(board);

  if (isMaximizing) {
    // AI turn — tối đa điểm
    let best = -Infinity;
    for (const move of getMoves(board)) {
      applyMove(board, move);
      const val = minimax(board, depth-1, false, alpha, beta);
      undoMove(board, move);

      best  = Math.max(best, val);
      alpha = Math.max(alpha, val);

      // Alpha-Beta pruning ✂️
      if (beta <= alpha) break; // cắt nhánh!
    }
    return best;

  } else {
    // Human turn — tối thiểu điểm
    let best = +Infinity;
    for (const move of getMoves(board)) {
      applyMove(board, move);
      const val = minimax(board, depth-1, true, alpha, beta);
      undoMove(board, move);

      best = Math.min(best, val);
      beta = Math.min(beta, val);
      if (beta <= alpha) break;
    }
    return best;
  }
}
Game Tree — Minimax Visual
            [ROOT] MAX (AI)
         ┌───────┴──────┐
      [B] MIN      [C] MIN
    ┌───┴───┐    ┌──┴──┐
  [D]+3 [E]-2  [F]+1 [G]+5✂
→ B chọn min(-2) = -2
→ G bị cắt (alpha-beta), không cần tính
→ ROOT chọn max(-2, ...) = tìm nước tốt nhất
⚡ Tại Sao Alpha-Beta Quan Trọng?
Minimax thuần: O(b^d) nodes — với cờ caro 15×15 và depth 5, con số khổng lồ. Alpha-Beta cắt ~50% nodes, giảm còn O(b^(d/2)). Depth 5 → tương đương depth 10 không có pruning.
Concepts áp dụng: Recursion, Game tree traversal, Pruning optimization. Nền tảng của Chess engines (Stockfish), AlphaGo và nhiều AI game thực tế.
🧠
Minimax là ví dụ tuyệt vời của recursion trong thực tế. Hiểu code này nghĩa là bạn đã sẵn sàng cho Computer Science và thuật toán nâng cao.
